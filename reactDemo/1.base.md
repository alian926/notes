## 基础
组件实例的三大核心属性(state, props, refs)(类组件)
``` js
class MyPage extends React.Component {
    // 构造器 对于一个 一个组件, 只调用一次
    constructor(props) {
        // super 由js语法规定,必须调用后才能使用constructor中的this
        super(props)
        // 初始化状态, state 必须是一个对象
        this.state = {
            name: 'Tom'
        }
    }
    // render 函数 在初始化和更新的时候都会调用 1+n次
    render() {
        // 解构, 读取状态
        let {name} = this.state
        return (
            <h1>This is {name}</h1>
        )
    }
}
ReactDOM.render(<MyPage>, document.getElementById('app'))
```

react中事件注册
``` js
class MyComponent extends React.Component {
    constructor() {
        super()
        // 这种方式,将原型上的方法,通过bind固定this,注册到实例上
        this.state = {
            title = '初始的'
        }
        // 实例上注册的方法
        this.selfClick = this.selfClick.bind(this)
    }
    // 实例上注册的方法
    handleClick = () => {
        // setState 会进行属性合并, 类似于 Object.assign()
        this.setState({
            title: '改变的'
        })
        console.log('clicked')
    }
    // 原型上注册的方法
    selfClick() {
        this.setState({
            title: '改变的'
        })
        console.log('clicked')
    }
    render() {
        return (
            <h1 onClick={this.handleClick}>按钮</h1>
            <h1 onClick={this.selfClick}>按钮2</h1>
        )
    }
}
```

精简代码内容
``` js
class Weather extends React.Component {
    // 初始化状态
    state = {
        isHot: true
    }
    // 自定义方法
    handleClick = () => {
        let { isHot } = this.state
        this.setState({
            isHot: !isHot
        })
    }
    render() {
        let text = this.state.isHot ? 'hot' : 'cold'
        return (
            <h1 onClick={this.handleClick}>This is {text}</h1>
        )
    }
}
```

props 约束
``` js
// 这种格式在16版本以后不再使用
Person.propTypes = {
    name: React.PropTypes.string,
    age: React.PropTypes.number,
}
// 16以后 prop-types库中引入, 用于对组件标签属性限制
Person.propTypes = {
    name: PropTypes.string.isRequired,
    age: PropTypes.number,
    // function是一个关键字, 所以要使用简写func
    speak: PropTypes.func
}
Person.defaultProps = {
    name: 'Bob',
    age: 18
}
```

props 约束的简写, 或者说移动到类内部来写
``` js
class Person extends React.Component {
    // 标签属性的限制
    static propTypes = {
        name: PropTypes.string.isRequired,
        age: PropTypes.number,
        speak: PropTypes.func
    }
    // 默认值
    static defaultProps = {
        name: 'Bob',
        age: 18
    }
}
```

constructor
``` js
// 通常React中, constructor 通常用于以下两种情况
// 1.通过给this.state赋值对象初始化内部state
// 2.为事件处理函数绑定this
// 所以在 高版本的 js 中,可以省略constructor
constructor(props) {
    super(props)
    // 是否接收props,传递给super, 是否希望在constructor中使用this.props
    // 可以直接使用props呀
    console.log(this.props)
}
```

``` js
class Demo extends React.Component {
    state = {
        inputValue: '',
    }
    handleInput = (e) => {
        let val = e.target.value
        this.setState({
            inputValue: val
        })
    }
    handleAlert =() => {
        alert(this.refs.input1.value)
    }
    render() {
        return (
            <div>
                <input type="text" onInput={this.handleInput} value={this.state.inputValue}/>
                <button onClick={this.handleAlert}>点击</button>
            </div>
        )
    }
}
```
